저는 GitHub 사용법에 대해 GitHub 계정 생성부터 리포지토리 클론 방법, 브랜치 생성 및 병합 과정, 커밋 메시지 작성 규칙까지 단계별로 세부적으로 교육하였습니다.
 브랜치를 명확히 하여 팀원들이 각자의 브랜치에서 작업하도록 하고, main 브랜치와의 병합 시 Pull Request를 생성하여 작업 내용을 팀원들과 공유하고 확인할 수 있도록 했습니다.
팀원이 동일한 파일을 수정했을 때 충돌이 발생하는 어려움이 있었으나, 이를 해결하기 위해 팀원들이 실시간으로 GitBoard에서 작업 상황을 관리하고, 작업 중복을 방지하여 충돌을 최소화했습니다.
 Git 충돌을 방지하기 위해 병합 전 충분히 소통하였고, 중간에 발생할 수 있는 충돌을 최소화하고자 노력하였습니다.
그 외에도 캔버스를 이용한 계절별 날씨 애니메이션 효과를 통하여 방문자가 지루하지 않도록 노력하였습니다.
 개성이 강한 팀원들로 모여 의견 충돌이 많았지만, 서로의 의견을 존중하고 조율하며 갈등을 해결해 나갔습니다. 이러한 과정에서 더 나은 팀워크를 이끌어낼 수 있었고, 협업의 중요성을 깊이 깨달았습니다.
앞으로도 이러한 경험을 바탕으로, 존중과 소통을 통해 더 나은 성과를 이끌어낼 수 있는 개발자가 되겠습니다.
